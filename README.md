# Inventory Management System

A full-stack inventory management application with natural language database querying powered by LangChain and OpenAI, featuring sales forecasting using Facebook Prophet.

## Overview

This system provides a comprehensive inventory management solution with the following features:

- **Product Management**: Track products, suppliers, and inventory levels
- **Sales Analytics**: Monitor sales transactions and daily demand patterns
- **Sales Forecasting**: Predict future sales using Prophet time series forecasting
- **Natural Language Queries**: Ask questions about your data in plain English using AI-powered SQL generation
- **Real-time Data Visualization**: View and filter data through interactive tables

## Tech Stack

### Backend

- **Flask**: Python web framework
- **SQLAlchemy**: ORM for database management
- **SQLite**: Database engine
- **LangChain**: AI-powered query generation
- **OpenAI GPT-4**: Natural language processing
- **Marshmallow**: Data serialization
- **Prophet**: Time series forecasting for sales prediction
- **Flask-Migrate**: Database migration management

### Frontend

- **React**: UI framework
- **react-data-table-component**: Interactive data tables

## Database Schema

The system includes seven main tables:

1. **products** - Product catalog with SKUs, pricing, and specifications
2. **suppliers** - Supplier contact information
3. **product_suppliers** - Product-supplier relationships with lead times
4. **sales** - Complete transaction history
5. **inventory** - Current stock levels and reorder points
6. **daily_demand** - Aggregated daily demand for forecasting
7. **future_sales** - Predicted sales forecasts generated by Prophet

## Setup Instructions

### Prerequisites

- Python 3.8+
- Node.js 14+
- OpenAI API key

### Backend Setup

1. **Install Python dependencies**:

```bash
cd backend/server
pip install -r requirements.txt
```

2. **Configure environment variables**:

   Create or update `backend/langchain/.env`:

```
OPENAI_API_KEY=your_openai_api_key_here
```

Update `backend/server/.env` if needed:

```
FLASK_APP=app.py
FLASK_ENV=development
SECRET_KEY=your_secret_key
```

3. **Generate sample data** (optional):

```bash
python backend/scripts/generate_inventory_data.py
```

4. **Seed the database**:

```bash
python backend/scripts/jsonintodb.py
```

5. **Run database migrations**:

```bash
flask db upgrade
```

6. **Generate sales forecasts** (optional):

```bash
python backend/scripts/futuresalesintodb.py
```

This will use Prophet to generate 100-day sales forecasts for all products based on historical data.

7. **Run the Flask server**:

```bash
npm run start-api
```

The backend will be available at `http://localhost:5001`

### Frontend Setup

1. **Install Node dependencies**:

```bash
npm install
```

2. **Start the React development server**:

```bash
npm start
```

The frontend will be available at `http://localhost:3000`

## Usage

### Viewing Data

- Use the navigation buttons to switch between different tables (Products, Suppliers, Sales, Inventory, Future Sales, etc.)
- Use the search box to filter table data in real-time
- Click column headers to sort data

### Natural Language Queries

The AI-powered search feature allows you to ask questions in plain English:

**Example queries**:

- "What is the current stock of Wireless Mouse?"
- "Show me all products with low inventory status"
- "What are the total sales for each product?"
- "Which products cost more than $100?"
- "List suppliers for Electronics category"
- "What are the predicted sales for next month?"

The system will:

1. Convert your question to SQL
2. Execute the query
3. Return a natural language answer

## API Endpoints

- `GET /products` - Retrieve all products
- `GET /suppliers` - Retrieve all suppliers
- `GET /product_suppliers` - Retrieve product-supplier relationships
- `GET /sales` - Retrieve sales transactions
- `GET /inventory` - Retrieve inventory levels
- `GET /daily_demand` - Retrieve daily demand data
- `GET /future_sales` - Retrieve predicted sales forecasts
- `POST /ask_db` - Submit natural language queries

## Sales Forecasting

The system uses Facebook Prophet to generate sales forecasts:

### How It Works

1. **Historical Analysis**: Prophet analyzes 2 years of historical sales data for each product
2. **Pattern Recognition**: Identifies trends, seasonality, and weekly patterns
3. **Future Predictions**: Generates 100-day forecasts with confidence intervals
4. **Per-Product Forecasts**: Individual forecasting models for each of the 28 products

### Forecast Features

- Daily demand predictions
- Seasonal trend analysis
- Confidence intervals (95% by default)
- Aggregated forecasts across all products

### Running Forecasts

```bash
cd backend/scripts
python futuresalesintodb.py
```

The forecast data is stored in the `future_sales` table and can be:
- Viewed through the "Future Sales" button in the UI
- Queried using natural language search
- Used for inventory planning and procurement decisions

## LangChain Integration

The system uses LangChain to:

- Parse natural language questions
- Generate appropriate SQL queries with JOINs when needed
- Execute queries against the SQLite database
- Format results into natural language responses

The LangChain agent understands:

- Table relationships and foreign keys
- Common business terminology
- Multi-table queries requiring JOINs
- Context about inventory, pricing, and stock levels
- Forecasting and prediction queries
